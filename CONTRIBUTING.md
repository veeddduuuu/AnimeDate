# Contributing to Waifu / AnimeDate

First off, thank you for considering contributing to Waifu! It's people like you that make this project better for everyone.

## Where to Start?

1. Check out the open issues for things we need help with.
2. If you find a bug, please create an issue first with detailed steps to reproduce it.
3. If you want to propose a new feature, open an issue to discuss it before you start coding.

## Development Workflow

1. Fork the repo and create your branch from `main`.
2. Ensure you have followed the setup instructions in `README.md`.
3. If you've added code that should be tested, add tests.
4. If you've changed APIs, update the documentation.
5. Ensure the test suite passes (if applicable).
6. Issue that pull request!

## Code Style

- Frontend: We use standard JavaScript/TypeScript practices along with Tailwind CSS. Follow the existing style.
- Backend: Python code should follow PEP 8. 

## Pull Request Process

1. Update the README.md with details of changes to the interface, this includes new environment variables, exposed ports, useful file locations, and container parameters.
2. The PR will be reviewed by maintainers, and we might request some changes before it can be merged.

Thank you for contributing!
